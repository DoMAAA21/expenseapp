# Full Stack Developer Exam - Technical Requirements

## Overview
- Create a `ReactJS` or `Flutter` app with high component reusability, proper state management, tests, and code standards.
- If applicable, create an API for the frontend app using one of the following:
  - `Golang`
  - `Node.js` (or any framework)
- You may use either a `NoSQL` or `SQL` database.
- You may use public APIs if you are not familiar with creating backend APIs (example: [https://swapi.dev/](https://swapi.dev/)).
- You will decide the application's functionality.
- Better quality and more purposeful features will earn more points.
- You will have **3 days** to complete the task.

## Objective
- Gauge frontend and backend skills under time pressure.
- Assess creativity in designing and building an application.

## Requirements
- Must have a **login page**.
- Must include **2 workflows or 4 screens** of your choosing.
  - The login page and login workflow are excluded from this count.
- Must run out of the box.
  - Failure to run the application(s) will result in automatic failure.

## Must-Haves
- Use `Git`.
- Include a `README.md`.
  - Must contain instructions on how to run the application.
- Demonstrate API integration (public API or self-built API).
- Use proper routing practices.
- Observe proper code structure.
- Add unit tests.

## Nice-to-Haves
- Dockerized app
- GraphQL implementation
- Push notifications
- Page analytics
